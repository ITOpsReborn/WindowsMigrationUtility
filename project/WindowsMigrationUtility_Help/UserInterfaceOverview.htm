User Interface Overview

On initial launch and your ScanState and/or LoadState syntax is blank it will notify you. You can select "Yes" to open up WMU's configuration settings.

WMU Main Form

The below is a brief overview of the options you have below.

System Type Information
You can run either a capture or restore on a local or remote system. Local system utilizes all built in file operations and remote systems rely on copying files and invoking commands via PSExec.
Select Your Option
This utility you can either capture or restore user and system information utilizing Microsoft's ® USMT utility.
USMT Options
You will see if you select the "Restore" option you have the option to specify a "Custom Decryption Key". This option is for captures that happened outside this utility. For an example you capture the system utilizing Configuration Manager, and for various reasons the restore doesn't work. You can get the encryption key from the database, and now specify the key inside the text box. Don't worry about the length the application creates a text file for it and uses the key file to decrypt.
Again if you select the "Restore" option you also have the option to utilize USMT Hard Linking. If you select this the custom decryption key section is disabled. There is no need for this option if you are restoring from a hard link capture.
USMT Path; for capture it will be where the USMT folder is created and then *.MIG file is captured to. For restore this is where the USMT folder is created, for an example "E:\WMU\" or "\\Server\WMU\" and the directory listing has folder inside ".\USMT\USMT.mig".
Log File Locations
The log files for ScanState and LoadState are listed in this area. The log files are always on the system where the process is being ran on.
